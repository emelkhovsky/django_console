Lesson 4 - Processes,
users, network
В этом уроке мы познакомим вас сразу с тремя видами команд: команды процессов, пользователей, сети. Для начала рассмотрим команды Linux для управления процессами:

#ps/grep

Команда ps выдает информацию об активных процессах. Выводятся идентификатор процесса, идентификатор терминала, истраченное к данному моменту время ЦП и имя команды.
Используя опцию «ps -e», можно увидеть информацию об определённом процессе.

Команда «ps – aux» выводит полную информацию о процессах.

Команда grep используется, чтобы найти конкретный процесс. Например, если ввести grep file, то команда grep выполнит поиск по результатам команды ps, и на экран будут выведены только те строки, которые содержат слово file.

Попробуйте ввести:

ps

Вы увидите информацию об активных процессах.

#kill/xkill/
pkill/killall

Эти команды служат для завершения процессов. Основное различие между ними — способ идентификации.

kill — прекращает процесс(посылает сигнал процессу), находя его через идентификатор процесса (PID).
Если вам нужно завершить операцию «process», вам необходимо ввести kill process.

xkill — графический способ. После ввода команды курсор превращается в крестик, который при клике на окно открытого процесса, останавливает его.

pkill — завершает конкретный процесс, находя его по имени.
killall — прекращает все процессы с указанным именем.

kill -l — выводит список сигналов и и числовые значения.

С помощью сигналов мы можем посылать сигналы процессам с определенным PID. Например, чтобы принудительно завершить процесс можно использовать команду kill -KILL PID. Эту же команду можно заменит на kill -9 PID, т.к. числовое значение сигнала KILL равно 9.

Попробуйте ввести:

kill -l

Вы увидите список сигналов.

#top

Эта команда, вызывающая консольную программу мониторинга активных процессов в системе.

Информация отображается в виде таблицы со списком процессов в реальном времени.

Попробуйте ввести:

top

#time

Команда-секундомер, позволяющая подсчитать время исполнения процесса. Результатом измерения станет блок данных с тремя важнейшими параметрами:

real — общее время исполнения;

user — сколько времени пользователь занял у CPU;
sys — сколько времени CPU было потрачено системой.

Попробуйте ввести:

time ls -l

В выводе вы увидите значения этих трех параметров.
Рассмотрим команды для управления пользователями. В этих уроках ими запрещено пользоваться в рамках этой консоли, но в своих личных проектах вы запросто сможете ими воспользоваться:)

#useradd/userdel/
usermod

Это команды управления учётными записями пользователей.
«useradd имя_пользователя» - создание нового пользователя;

«userdel имя_пользователя» - удаляет учётную запись и файлы пользователя;

«usermod [параметры] имя_пользователя» изменяет системные файлы учётных записей согласно переданным в командной строке параметрам;

#passwd

Обычный пользователь может поменять только пароль, связанный с его именем, введя passwd имя_пользователя.

В режиме суперпользователя можно сбросить все пользовательские пароли в системе.
И последний вид команд, который мы бы хотели вам показать в этом уроке – команды для управления сетью:

#ip

Утилита выводит полный список параметров для настройки работы с сетью. Среди них:

link — сетевое устройство;

address — IP-адрес сетевого устройства;

monitor — мониторинг устройства;

route — маршрутизация;

tunnel — туннелированные;

Синтаксис: ip [опции] объект команда [параметры]

Опции - это глобальные настройки, которые сказываются на работе всей утилиты независимо от других аргументов;

Объект - объект или устройство, с которым будем работать или о котором будем узнавать информацию;

Команды - какое-либо действие с объектом;

Команд с ip очень много, одна из них - ip address. ip address show - показать все ip адреса и их интерфейсы.

Попробуйте ввести:

ip address show

Вы сможете увидеть все ip адреса и их интерфейсы.

#ping

Эта команда проверяет наличие и качество интернет-подключения. В качестве адреса узла можно передавать как ip адрес, так и доменное имя.

Попробуйте ввести:

ping google.com

После ввода, если есть подключение, то вы увидите надпись, похожую на «64 bytes from lg-in-f138.1e100.net (64.233.165.138): icmp_seq=1 ttl=45 time=13.1 ms»

#nethogs

Утилита мониторинга сетевой активности устройств в системе. Если у вас медленный интернет, то вам, наверное, было бы интересно знать, сколько трафика использует какая-либо программа в Linux или какая программа потребляет всю скорость. Как раз с помощью этой утилиты можно это узнать.
Объект - объект или устройство, с которым будем работать или о котором будем узнавать информацию;

Команды - какое-либо действие с объектом;

Команд с ip очень много, одна из них - ip address. ip address show - показать все ip адреса и их интерфейсы.

Попробуйте ввести:

ip address show

Вы сможете увидеть все ip адреса и их интерфейсы.
